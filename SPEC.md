# BikeTracker — техническое задание

Простое приложение для трекинга велопоездок. Без соц-сетей, соревнований, планирования маршрутов и прочего лишнего.

## Стек

- Kotlin + Jetpack Compose
- Только Android
- FusedLocationProviderClient — GPS
- Foreground service — трекинг в фоне при выключенном экране
- osmdroid (OpenStreetMap) — карта, без ключей и аккаунтов
- Room — локальное хранение истории (SQLite), без облака

## Локализация

- Языки: английский (по умолчанию), украинский, русский. Все три ведём синхронно (переводы пишутся сразу при добавлении строки).
- Механизм — строковые ресурсы Android: `values/strings.xml` (en), `values-uk/`, `values-ru/`. Строки без пунктуации, пунктуация добавляется в коде.
- Выбор языка — в Настройках (per-app locales через системный LocaleManager, minSdk 33), плюс переключатель темы.

## Мини-справка по GPS (для тех, кто с этим не работал)

- Телефон периодически получает от спутников **точку**: широта, долгота, **accuracy** (радиус погрешности в метрах — чем меньше, тем точнее), скорость, время.
- Точки идут не идеально ровно: возможны шумы (в туннелях, среди высоток, при слабом сигнале) — точка может "прыгнуть" в сторону на десятки метров.
- Расстояние считается как сумма отрезков между последовательными точками (формула гаверсинуса по координатам).
- Чтобы не накручивало из-за шума, точки с плохим accuracy или нереальным скачком скорости отбрасываются перед тем как попасть в трек (это и есть "фильтр по accuracy и скорости").
- На Android для GPS в реальном времени нужно разрешение `ACCESS_FINE_LOCATION` (точная геолокация, "при использовании приложения"). `ACCESS_BACKGROUND_LOCATION` не требуется: foreground service типа location, запущенный из видимого экрана, продолжает получать координаты при свёрнутом приложении и выключенном экране.
- Фоновый трекинг на Android обязан идти через **foreground service** с постоянным уведомлением ("Идёт запись поездки: 5.2 км") — так требует система, иначе ОС может убить процесс для экономии батареи.

## Экраны

Навигация: нижний бар с 4 вкладками + верхний бар (заголовок экрана + кнопки, напр. "Выход").

1. **Главный (трекинг)** — текущая скорость, дистанция, время поездки, высота (по GPS), средняя скорость за поездку, часы. Кнопки: старт / пауза (ручная) / стоп-save (одна кнопка — останавливает и сразу сохраняет в историю; ненужную поездку проще потом удалить из истории, чем городить отдельное подтверждение).
2. **Карта** — карта с текущим треком поездки (можно посмотреть на ходу, остановившись).
3. **История** — сверху сводная статистика (суммарный километраж и время за неделю/месяц/год/всё время), ниже список завершённых поездок (дата, дистанция, время, средняя скорость), тап открывает детали (карта + полная статистика поездки, включая макс. скорость + кнопка экспорта в GPX), можно удалить поездку.
4. **Настройки** — язык (EN/UK/RU), тема (светлая/тёмная/системная), единицы измерения (закреплено км/км/ч), порог авто-паузы, порог авто-сохранения при длительной паузе (по умолчанию 15 мин, настраивается).

**Выход** — кнопка в верхнем баре. Если есть активный трекинг — предупреждение и блокировка выхода, пока поездка не завершена.

**Авто-сохранение при долгой паузе** — если поездка на паузе (ручной или авто) дольше настроенного порога (по умолчанию 15 минут), она автоматически завершается и сохраняется в историю — на случай, если забыли остановить трекинг.

**Высота по GPS** — учитывать, что точность вертикали хуже горизонтали (спутниковый GPS: погрешность ~10-30 м). Если у устройства есть барометр — можно рассмотреть уточнение позже, вне первой версии.

**Уличная эргономика** — приложение используется на велосипеде под ярким солнцем: крупные цифры, крупные кнопки с заливкой (не outlined), высокий контраст. Экран не гаснет, пока идёт активная поездка (запись/пауза). Полноценный высококонтрастный вид доводится вместе с переключателем темы.

## Данные

- **Trip** (поездка): id, дата/время начала, дата/время окончания, дистанция (км), длительность (без учёта пауз), средняя скорость, макс. скорость.
- **TrackPoint** (точка трека): tripId, широта, долгота, время, скорость.

## Функциональность трекинга

- Старт / ручная пауза (возобновление) / стоп-save (одна кнопка, останавливает и сохраняет).
- Авто-пауза: при падении скорости ниже порога (~2 км/ч) на несколько секунд трек автоматически ставится на паузу; возобновляется при начале движения.
- Авто-сохранение: если поездка на паузе дольше настроенного порога (по умолчанию 15 мин) — завершается и сохраняется автоматически.
- Фильтрация GPS-точек по accuracy и скорости перед добавлением в трек.
- Работает в фоне (foreground service), при выключенном экране.

## Статистика и экспорт

- **Макс. скорость** за поездку — вычисляется из точек трека, показывается в live-статистике и в деталях поездки.
- **Сводная статистика** — суммарный километраж и время за неделю / месяц / год / всё время, на экране История сверху.
- **Экспорт в GPX** — из деталей поездки, стандартный формат (можно открыть на компьютере и в других приложениях). Данные пользователя остаются с пользователем; не облако, не публикация.

## Явно вне рамок

- Соц-сети, публикация поездок, лента
- Соревнования, достижения, лидерборды
- Планирование маршрутов, навигация, турн-бай-турн
- Облачная синхронизация
